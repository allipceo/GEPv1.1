## 3. statsStore 통계 카운팅 흐름

### 저장소 위치
- **파일:** `src/stores/statsStore.js` + `src/utils/statsStorage.js`
- **localStorage 키:** `'gep_stats_v1'` (statsStorage.js line 18)
- **버전:** `'1.0'` (statsStorage.js line 19)
- **앱 시작 시 초기화:** `stats: loadStats()` (statsStore.js line 17)

### 스키마 구조
```js
{
  version:   "1.0",
  total:     { solved: number, correct: number },           // 누적 전체
  daily:     { "YYYY-MM-DD": { solved, correct } },         // 날짜별 (자동 키)
  bySubject: { "보험업법": { solved, correct }, ... },      // 세부과목별 (12개)
  byRound:   { 23: { solved, correct }, ... }               // 회차별
}
```

### 카운팅 트리거 — Question.jsx handleAnswer() (line 106-120)
```js
const handleAnswer = (num) => {
  saveAnswer(question.id, num)                              // examStore 답안 저장
  setLocalAnswered(prev => new Set([...prev, question.id])) // UI 상태 업데이트

  if (!recordedSet.has(question.id)) {                      // line 111: 중복 방지
    updateStats({
      subject: question.subSubject,                         // 세부과목 (예: '보험업법')
      round:   selectedRound,                               // 회차 (예: 23 또는 '전체')
      solved:  1,
      correct: num === question.answer ? 1 : 0,             // 정답 여부
    })
    setRecordedSet(prev => new Set([...prev, question.id])) // 기록 완료 표시
  }
}
```

### 중복 방지 메커니즘 — 2개의 Set 분리

| Set | 정의 위치 | 역할 | 이전 버튼 클릭 시 |
|-----|----------|------|-----------------|
| `localAnswered` | Question.jsx line 49 | UI 답안 표시 여부 | 이동 대상 문제 ID **제거** |
| `recordedSet` | Question.jsx line 52 | 통계 중복 기록 방지 | **유지** (통계는 한 번만) |

**이전 버튼 동작** (Question.jsx line 122-133):
```js
const handlePrev = () => {
  const prevId = filteredQuestions[currentIndex - 1]?.id
  if (prevId) {
    setLocalAnswered(prev => {              // localAnswered에서만 제거
      const next = new Set(prev)
      next.delete(prevId)
      return next
    })
    // recordedSet은 수정하지 않음 → 돌아가서 재답변해도 통계 재기록 안됨
  }
  setCurrentIndex(currentIndex - 1)
}
```

### updateStats 내부 처리 (statsStore.js line 27-66)
```js
updateStats: ({ subject, round, solved, correct }) => {
  const today   = todayKey()  // "YYYY-MM-DD" (statsStorage.js line 57-59)

  // 4개 버킷 동시 누적
  total:     prev.total.solved  + solved           // line 37-40
  daily:     prev.daily[today]  + solved/correct   // line 41-47 (날짜 자동 키)
  bySubject: prev.bySubject[subject] + ...         // line 48-54 (세부과목 키)
  byRound:   prev.byRound[round]    + ...          // line 55-61 (회차 키)

  set({ stats: newStats })
  saveStats(newStats)  // 즉시 localStorage 동기 저장 (line 65)
}
```

### 금일 통계 자정 자동 리셋
- 별도 리셋 로직 없음. `todayKey()`가 날짜 변경 시 새 키 반환.
- 예: 2026-02-21 → 2026-02-22로 날짜 바뀌면 `daily["2026-02-22"]` 신규 생성, `default { solved:0, correct:0 }`.
- **과거 날짜 데이터는 삭제되지 않고 daily 객체에 누적** (용량 주의).

## 5. Known Issues

### Issue-1: byRound['전체'] 혼합 집계
**현상:** 과목별 학습 모드(round='전체')로 풀면, `statsStore.byRound['전체']` 키에 누적됨.
```js
// statsStore.js line 55-61
byRound: {
  ...prev.byRound,
  ['전체']: { solved: N, correct: M }  // ← 숫자 키가 아닌 문자열
}
```
**영향:** `byRound` 조회 시 `'전체'` 버킷이 따로 생성됨. 현재 UI에서 byRound 조회 기능이 없으므로 사용자 노출 없음.
**해결 방향:** 통계 기록 시 `round='전체'`면 실제 `question.round` 값을 사용하거나 byRound 기록 스킵.

### Issue-2: daily 버킷 무한 누적
**현상:** 날짜별 daily 키가 삭제되지 않고 누적됨.
**영향:** 장기 사용 시 localStorage 용량 증가. 현재 문제 없음 (1일 1~2 KB 수준).
**해결 방향:** 최근 N일 이외 데이터 정리 로직 추가.

### Issue-3: setSubSubject 항상 currentIndex=0 리셋
**현상:** `setSubSubject` 호출 시 progressMap 미조회, 무조건 0으로 리셋 (examStore.js line 154).
**영향:** 과목별 학습 탭 → 세부과목 클릭 → 항상 1번 문제부터 시작.
**현재 의도:** setSubSubject는 세부과목 '변경' 개념이므로 0 리셋이 UX상 자연스러움.

### Issue-4: showResume 미사용 (Home.jsx)
**현상:** `showResume` 변수 계산됨(line 130-133)이나 이어풀기 버튼 표시/숨김에 사용 안됨.
```js
const showResume = isReady && filteredQuestions.length > 0 && (resumeIndex > 0 || ...)
// 단, 이 값으로 버튼을 조건부 렌더링하지 않음
```
**영향:** 미사용 변수 (dead code). 기능 이상 없음.
