## 1. 작업 목적

## 2. 원인 분석

### 데이터 구조 비교

| 회차 | 법령 | 손보1부 | 손보2부 | 상태 |
|------|------|---------|---------|------|
| 23~30회 | 40문제 | 40문제 | 40문제 | ✅ 정상 |
| **31회** | **120문제** | **0문제** | **0문제** | **❌ 버그** |

### 버그 패턴

31회 120문제 전체의 `subject` 필드가 `'법령'`으로 잘못 설정됨.

**ID 필드의 과목 코드 (T1/T2/T3):**
```
IBEX_31_001_T1_S01_001 → T1 = 법령
IBEX_31_001_T2_S05_041 → T2 = 손보1부
IBEX_31_001_T3_S09_081 → T3 = 손보2부
```

다른 회차(23회 예시):
```
IBEX_23_001_T1_S01_001 → subject: '법령'    ✅
IBEX_23_001_T2_S05_041 → subject: '손보1부' ✅
IBEX_23_001_T3_S09_081 → subject: '손보2부' ✅
```

### 증상

| 시나리오 | 증상 |
|----------|------|
| 31회 1교시(part1) | 120문제 표시 (40문제여야 함) |
| 31회 2교시(part2) | 0문제 → LoadingView 무한 표시 → **"문제가 안뜹니다"** |

### MockExamQuiz 필터 로직

```javascript
// part1
filtered = allQuestions.filter(q => q.round === 31 && q.subject === '법령')
// 수정 전: 120개 반환 (손보 문제까지 포함)
// 수정 후: 40개 반환 ✅

// part2
filtered = [...손보1부40개, ...손보2부40개]
// 수정 전: 0개 반환 → LoadingView 무한
// 수정 후: 80개 반환 ✅
```

## 4. 주요 변경사항

### 수정 기준 (ID 필드의 T-prefix 활용)

```javascript
// T2 포함 → 손보1부 (roundNumber -40 조정)
if (q.id.includes('_T2_')) {
  return { ...q, subject: '손보1부', roundNumber: q.roundNumber - 40 }
}
// T3 포함 → 손보2부 (roundNumber -80 조정)
if (q.id.includes('_T3_')) {
  return { ...q, subject: '손보2부', roundNumber: q.roundNumber - 80 }
}
```
