# GEP_036_노팀장스킬활용가이드

**작성일시:** 2026.02.22
**작성자:** 고팀장 (Claude Code)
**대상:** 노팀장 (claude.ai) — 향후 지시문 작성 시 참고

---

## 개요

GEPv1.1 개발 효율화를 위해 Claude Code 스킬 4종을 `.claude/commands/`에 등록했습니다.
노팀장이 고팀장에게 지시문을 작성할 때 아래 스킬 호출 구문을 포함하면
반복 작업이 자동화되어 토큰 절감 + 일관성 유지가 가능합니다.

---

## 등록된 스킬 4종

| 스킬 | 호출 | 역할 |
|------|------|------|
| gep-deploy | `/gep-deploy` | 빌드 → 커밋 → push 자동화 |
| gep-doc | `/gep-doc` | GEP_XXX 작업 문서 자동 생성 |
| gep-report | `/gep-report` | 노팀장 보고 형식 즉시 출력 |
| gep-verify | `/gep-verify` | Node.js 검증 스크립트 생성+실행 |

## 노팀장 지시문 작성 권장 패턴

### ✅ 권장 — 스킬 명시 패턴

```
CLAUDE.md를 읽고 아래 작업을 진행하세요.

## 작업명: GEP_XXX_작업명

## 작업 내용
[구체적인 지시]

## 완료 후
1. /gep-verify  ← 로직 검증 (해당 시)
2. /gep-deploy  ← 빌드+커밋+push
3. /gep-doc     ← 작업 문서 생성
4. /gep-report  ← 보고 형식 출력
```

### ❌ 비권장 — 매번 수동 지시

```
# 이렇게 하지 마세요
완료 후 npm run build하고, git add하고, commit하고,
push하고, docs/GEP_XXX.md 만들고, 보고해주세요...
```

---

## 스킬별 사용 타이밍 가이드

### `/gep-verify` — 이럴 때 사용

```
## 완료 조건
1. /gep-verify 실행 → 전체 PASS 확인
```

- 로직 변경이 포함된 작업 (handleAnswer, filter, updateStats 등)
- 버그 수정 후 회귀 테스트 필요 시
- localStorage 관련 데이터 구조 변경 시
- **생략 가능:** 단순 UI 변경, 주석 추가, 문서 작업

---

### `/gep-deploy` — 항상 사용

```
## 완료 후
/gep-deploy
```

- **모든 작업에 필수** (빌드 확인 + 배포 포함)
- 빌드 실패 시 자동으로 중단하고 보고

---

### `/gep-doc` — 항상 사용

```
## 완료 후
/gep-deploy → /gep-doc
```

- **모든 작업에 필수** (CLAUDE.md 문서화 규칙)
- `/gep-deploy` 이후 호출 권장 (빌드 결과 포함해서 문서 작성)

---

### `/gep-report` — 보고 필요 시 사용

```
## 완료 후
/gep-report  ← 노팀장 창에 붙여넣기용
```

- 조대표님 최종 확인이 필요한 STEP 완료 시
- STEP 간 핸드오프 시 (STEP1 완료 → STEP2 승인 요청)

## 표준 지시문 템플릿 (노팀장 복사 사용용)

```markdown
CLAUDE.md를 읽고 아래 작업을 진행하세요.

## 작업명: GEP_XXX_[작업명]

## 배경
[왜 이 작업이 필요한지]

## 수정 대상
파일: [파일명]
위치: [함수/컴포넌트명]

## 변경 내용
[현재]
[코드]

[변경 후]
[코드]

## 완료 조건
1. [검증 항목 1]
2. [검증 항목 2]
3. /gep-verify (로직 변경 시)
4. /gep-deploy
5. /gep-doc
6. /gep-report

## 주의사항
- [수정 금지 파일]
- [기타 주의사항]
```

**스킬 적용 방식:**
```
완료 조건:
4. /gep-verify  (byRound 오염 검증)
5. /gep-deploy
6. /gep-doc
7. /gep-report
```

## 기대 효과

| 항목 | 기존 | 스킬 적용 |
|------|------|----------|
| 지시문 분량 | 길고 반복적 | 핵심만 간결하게 |
| 보고 형식 일관성 | 매번 다름 | 항상 동일 |

| 배포 절차 오류 | 가능 | 체크리스트 자동화 |
| 토큰 절감 | - | 지시문당 약 30~40% |
